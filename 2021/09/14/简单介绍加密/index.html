介绍加密知识(第一章)

简单介绍加密

1. 为什么要加密

现代生活中网络扮演着很重要的角色，没有网络我们几乎无法正常生活。所以网络上也传输着很多敏感信息，例如身份证号码、银行卡密码、各个网站的账户数据等等。在互联网发展的初期，这些数据很多都是明文传输，很容易被有心人截获利用。理所当然的，数据加密就成为了互联网一个必不可少的功能。

2. 加密的方法

2.1 对称加密

举一个最简单的例子。假设你要给朋友写信，信里的内容只有一个词 hello，并且你不想其他人知道信件的内容。投送过程不可能是安全的，任何人都有可能看到你的信件，怎样才能保证你的信件内容不会泄露呢？

你首先想到的可能是封条，只要保证封条完整，就说明信息没有泄露。
首先可以利用封条，只要保证封条完整，就说明信息没有泄露。问题是，封条可以被伪造。如何做到就算别人看到了信件内容也无法知道你要传递的信息呢？这就需要加密。

我们可以设定一个K值，所有字母都右移K位。例如 K＝4，hello 就会变成 lipps。如果不知道K的值，你就无法知道信件的真实内容。这就是对称加密，K就是密钥，加密成功与否取决于密钥只有发信人与收信人知道。
但这也有一个致命的缺点，如果别人破解了密钥，加密就失效了。就像二战时期图灵破解了德国的加密电报，获知了德国潜艇的部署情况。极大改变了二战的战局。
这就引申出另一个问题，如何传递密钥？

2.2 非对称加密

对称加密的前提是密钥不可泄露，那如何安全的传递密钥呢？
我们可以不通过网络传递密钥，比如亲自送到对方手上，但这样的成本太高，所以我们可以使用非对称加密。

非对称加密有一对密钥，分别是公钥和私钥。私钥由个人保管，公钥可以发布到网上。公钥加密的内容只能用私钥解密，相反，私钥加密的内容只能用公钥解密。使用非对称加密通信前需要交换公钥，双方发送到内容使用对方的公钥加密。所以每个人收到的内容只能用各自的私钥解密，而私钥是保密的，这就保证了信息的安全性。

3. 用一个例子介绍加密

我们用一个案例，访问百度，来介绍加密如何在实际生活中应用的。

首先，我们讨论一下在浏览器输入网址 https://www.baidu.com 后会发生什么。

  1. DNS 域名解析（域名解析成ip地址，走UDP协议，因此不会有握手过程）：浏览器将 URL 解析出相对应的服务器的 IP 地址（1. 本地浏览器的 DNS 缓存中查找 2. 再向系统DNS缓存发送查询请求 3. 再向路由器DNS缓存 4. 网络运营商DNS缓存 5. 递归搜索），并从 url 中解析出端口号
  2. 浏览器与目标服务器建立一条 TCP 连接（三次握手）
  3. 浏览器向服务器发送一条 HTTP 请求报文
  4. 服务器返回给浏览器一条 HTTP 响应报文
  5. 浏览器进行渲染
  6. 关闭 TCP 连接

完成上面的步骤之后，HTTPS在传输数据之前需要客户端（浏览器）与服务端（网站）之间进行一次握手，在握手过程中将确立双方加密传输数据的密码信息。TLS/SSL协议不仅仅是一套加密传输的协议，TLS/SSL中使用了非对称加密，对称加密以及 HASH 算法。
握手过程的简单描述如下：

  1. 浏览器将自己支持的一套加密规则发送给网站。
  2. 网站从中选出一组加密算法与 HASH 算法，并将自己的身份信息以证书的形式发回给浏览器。证书里面包含了网站地址，加密公钥，以及证书的颁发机构等信息。
  3. 获得网站证书之后浏览器要做以下工作：
    a. 验证证书的合法性（颁发证书的机构是否合法，证书中包含的网站地址是否与正在访问的地址一致等），如果证书受信任，则浏览器栏里面会显示一个小锁头，否则会给出证书不受信的提示。
    b. 如果证书受信任，或者是用户接受了不受信的证书，浏览器会生成一串随机数的密码，并用证书中提供的公钥加密。
    c. 使用约定好的 HASH 计算握手消息，并使用生成的随机数对消息进行加密，最后将之前生成的所有信息发送给网站。
  4. 网站接收浏览器发来的数据之后要做以下的操作：
    a. 使用自己的私钥将信息解密取出密码，使用密码解密浏览器发来的握手消息，并验证HASH是否与浏览器发来的一致。
    b. 使用密码加密一段握手消息，发送给浏览器。
  5. 浏览器解密并计算握手消息的HASH，如果与服务端发来的HASH一致，此时握手过程结束，之后所有的通信数据将由之前浏览器生成的随机密码并利用对称加密算法进行加密。

从上面的过程可以看出，浏览器生成的密码是整个数据加密的关键，非对称加密只在握手过程使用，把浏览器生成的密码发送到服务器。握手结束后所有信息使用该密钥进行加密。
